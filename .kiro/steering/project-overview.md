# MicroRealEstate (MRE) ‚Äî Project Overview

MicroRealEstate is an open-source property management application for landlords. It follows a microservices architecture, containerized with Docker, and uses a Yarn 3 monorepo.

## Workspace Packages (Yarn Workspaces)

All packages are scoped under `@microrealestate/*`:
- `@microrealestate/types` ‚Äî shared TypeScript types
- `@microrealestate/common` ‚Äî shared backend library
- `@microrealestate/gateway`, `api`, `tenantapi`, `authenticator`, `emailer`, `pdfgenerator`, `resetservice` ‚Äî services
- `@microrealestate/commonui`, `landlord`, `tenant` ‚Äî frontend apps
- `@microrealestate/e2e` ‚Äî end-to-end tests
- `@microrealestate/cli` ‚Äî CLI tool (not published, used internally)

## Key Commands

- `yarn dev` ‚Äî Start all services in development mode (Docker + hot reload)
- `yarn build` ‚Äî Build all Docker images for production
- `yarn start` ‚Äî Start in production mode
- `yarn stop` ‚Äî Stop all containers
- `yarn ci` ‚Äî Start in CI mode (for automated testing)
- `yarn lint` ‚Äî Lint all workspaces
- `yarn format` ‚Äî Format all workspaces with Prettier
- `yarn e2e:ci` / `yarn e2e:run` / `yarn e2e:open` ‚Äî Run Cypress e2e tests
- `yarn deploy:nas` ‚Äî Merge master ‚Üí nas, push (GHCR builds `:nas` images), optionally trigger Portainer redeploy on the Synology NAS (self-hosted fork only)

## Branches

- `master` ‚Äî mirrors upstream in spirit (same source code), used for local dev. Authorship on commits was rewritten for the fork so git sees no common ancestor with `microrealestate/microrealestate` anymore ‚Äî upstream changes can only be pulled via `git cherry-pick`, not `git merge`.
- `nas` ‚Äî self-hosted deployment branch for the directorscut82 fork. Adds 3 source changes for multi-origin support (LAN + Tailscale) and `.github/workflows/nas-ci.yml` which builds `:nas` + `:nas-<sha>` images to GHCR. The `docker-compose.nas.yml` stack file is local-only (gitignored) and never pushed. Deployment is automated via `scripts/deploy-nas.sh` (invoked by `yarn deploy:nas`). See `documentation/DEV_AND_DEPLOY.md` for the full workflow.

## Environment Configuration

- `base.env` ‚Äî Default values, versioned in git
- `.env` ‚Äî Local overrides with secrets (generated by `mre configure` CLI)
- Environment variables are merged: base.env values are overridden by .env values
- Secrets include: `REDIS_PASSWORD`, `CIPHER_KEY`, `CIPHER_IV_KEY`, `AUTHENTICATOR_*_TOKEN_SECRET`
